6. **Task - Fix mid-word boolean and URL port highlighting** (v1.0.7): Fixed gitconfig parser incorrectly highlighting "on" inside words and port numbers in URLs<br>
   **Result**: The gitconfig `StreamParser` had two tokenization bugs. First, `\b(true|false|yes|no|on|off)\b` regex matched "on" inside words like "extension" because `stream.next()` consumed characters one at a time, so after consuming "extensi" the next `token()` call saw "on" at position start where `\b` matched. Fixed by adding a lookbehind guard checking `stream.string.charAt(stream.pos - 1)` is not alphanumeric, plus lookahead `(?=[^a-zA-Z0-9_-]|$)` to ensure the boolean is followed by a non-word character or EOL. Second, `:8443` in URLs like `https://gitlab.srv.stellars-tech.eu:8443/path` was split into `:` (string) and `8443` (number) because the unquoted value chunk regex `[a-zA-Z0-9_./-]+` didn't include `:` or `@`. Expanded to `[a-zA-Z0-9_./:@~-]+` so URLs and git addresses are consumed as single string tokens. Removed unused `BOOLEAN_VALUES` Set. Added `string` property to `MockStringStream` test helper. Added two regression tests (tests 21-22) covering both bugs. All 22 tests pass. Published v1.0.7 to npm and PyPI.
